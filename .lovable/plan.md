

## পরিকল্পনা: অ্যাডমিন সেটিংসে Drag & Drop সেকশন অর্ডারিং

### বর্তমান অবস্থা
হোমপেজে ৫টি সেকশন হার্ডকোডেড অর্ডারে আছে: Hero → Features → Course → CTA → WhatsApp

### যা করা হবে

**ধাপ ১: ডাটাবেজে সেকশন অর্ডার সেভ করা**
- `settings` টেবিলে `section_order` কী যোগ করা
- ভ্যালু হবে JSON array: `["hero","features","course","cta","whatsapp"]`

**ধাপ ২: অ্যাডমিন সেটিংসে Drag & Drop UI**
- HTML5 native drag and drop ব্যবহার করে (কোনো নতুন লাইব্রেরি লাগবে না)
- নতুন কম্পোনেন্ট `SectionOrderSettings.tsx` তৈরি
- প্রতিটি সেকশন একটি ড্র্যাগেবল কার্ড হিসেবে দেখাবে (grip icon সহ)
- ড্র্যাগ করে উপরে-নিচে নিলে অর্ডার পরিবর্তন হবে
- সেভ বাটনে ক্লিক করলে `section_order` কী-তে JSON সেভ হবে

**ধাপ ৩: Index.tsx আপডেট**
- সেটিংস থেকে `section_order` ফেচ করা
- প্রতিটি সেকশনকে আলাদা কম্পোনেন্ট/ফাংশনে ভাগ করা
- `section_order` অনুযায়ী ডাইনামিক্যালি রেন্ডার করা (ফলব্যাক: ডিফল্ট অর্ডার)

### প্রযুক্তিগত বিস্তারিত

```text
Admin Settings Page:
┌─────────────────────────┐
│ ☰  হিরো সেকশন          │  ← drag handle
├─────────────────────────┤
│ ☰  ফিচার সেকশন         │
├─────────────────────────┤
│ ☰  কোর্স সেকশন         │
├─────────────────────────┤
│ ☰  CTA সেকশন           │
├─────────────────────────┤
│ ☰  WhatsApp সেকশন      │
└─────────────────────────┘
```

- সেকশন ম্যাপ: `{ hero: <HeroSection/>, features: <FeaturesSection/>, ... }`
- অর্ডার JSON: `["hero","features","course","cta","whatsapp"]`
- ড্র্যাগ করলে array reorder হবে, সেভে DB-তে যাবে
